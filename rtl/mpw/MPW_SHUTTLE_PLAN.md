# MPW Shuttle Plan — HCPU Silicon Implementation

## Overview

This document outlines the plan for taping out the HCPU (Hijaiyyah Core Processing Unit)
as an ASIC via Multi-Project Wafer shuttle services.

## Priority Order

| Priority | Program | Process | Cost | Die Area |
|---|---|---|---|---|
| **#1** | Efabless Open MPW (Google-sponsored) | SkyWater SKY130 (130nm) | Free (if accepted) | ~1 mm² |
| **#2** | TinyTapeout | SkyWater SKY130 | ~$300/tile | Limited |
| **#3** | Europractice | TSMC 130nm / 65nm | $5K–$20K | Custom |

## Prerequisite: FPGA Validation

> **CRITICAL**: HCPU must be fully verified on FPGA (Tang Nano 9K or Arty A7)
> before any MPW submission. This includes:
> - All 28 ROM entries verified
> - All guard checks passing on hardware
> - HCADD string integral verified (بسم test)
> - UART output matching HCVM software reference
> - Branch/loop programs executing correctly

## Option 1: Efabless Open MPW (SKY130)

### Architecture

```
┌──────────────────────────────────────────────────┐
│                  Caravel SoC                      │
│  ┌─────────────┐  ┌──────────────────────────┐   │
│  │ Management   │  │     User Project Area    │   │
│  │ SoC (RISC-V) │  │  ┌──────────────────┐   │   │
│  │             │  │  │     HCPU Core     │   │   │
│  │  - Boot ROM  │  │  │  ┌────────────┐  │   │   │
│  │  - SPI Flash │  │  │  │  Pipeline   │  │   │   │
│  │  - GPIO      │  │  │  │  F→D→E→M→W │  │   │   │
│  │             │  │  │  ├────────────┤  │   │   │
│  │             │  │  │  │ Codex ALU  │  │   │   │
│  │             │  │  │  │ Guard Unit │  │   │   │
│  │             │  │  │  │ ROM 252B   │  │   │   │
│  │             │  │  │  ├────────────┤  │   │   │
│  │             │  │  │  │ UART TX    │  │   │   │
│  │             │  │  │  └────────────┘  │   │   │
│  │             │  │  └──────────────────┘   │   │
│  └─────────────┘  └──────────────────────────┘   │
│                     Wishbone Bus                  │
└──────────────────────────────────────────────────┘
```

### Integration Requirements

1. **Wishbone bus adapter** for HCPU ↔ Caravel communication
2. **Clock domain**: Use Caravel's `wb_clk_i` (typically 40 MHz)
3. **Reset**: Wire to Caravel's `wb_rst_i`
4. **I/O mapping**:
   - GPIO[0] = UART TX
   - GPIO[1] = Guard LED
   - GPIO[2] = Halted indicator
5. **Instruction loading**: Via Wishbone writes to code RAM
6. **Status readback**: Via Wishbone reads from status register

### Submission Requirements

- Verilog RTL (already done)
- GDS II layout (generated by OpenLane flow)
- Comprehensive testbench (already done)
- Timing constraints
- Power analysis

### Timeline

| Phase | Duration | Description |
|---|---|---|
| FPGA validation | 2–4 weeks | Complete bring-up on Tang Nano 9K |
| Wishbone adapter | 1–2 weeks | Write `hcpu_wb_adapter.v` |
| OpenLane synthesis | 1–2 weeks | Clock tree, P&R, DRC/LVS |
| Submission | 1 week | Apply to Open MPW shuttle |
| Fabrication | 3–6 months | SkyWater fab processing |
| Testing | 2–4 weeks | Bring-up on Caravel eval board |

## Option 2: TinyTapeout

### Constraints

- Maximum **2×2 tiles** recommended for HCPU
- Each tile: 161 × 225 μm (~100 logic cells)
- I/O: 8 inputs + 8 outputs + 8 bidirectional
- Clock: External, up to ~50 MHz

### HCPU-min Design for TinyTapeout

Due to area constraints, a **reduced HCPU** would be needed:

| Feature | Full HCPU | HCPU-min |
|---|---|---|
| GPR | 18 × 32-bit | 4 × 8-bit |
| H-Reg | 16 × 144-bit | 2 × 144-bit |
| Pipeline | 5-stage | Single-cycle |
| ROM | 28 entries | 28 entries (required) |
| Stack | 256 entries | 8 entries |
| Instructions | 20+ | 8 (HLOAD, HCADD, HGRD, HALT, MOV, JMP, JEQ, PRINT) |

### I/O Mapping (8+8+8)

```
Inputs (8):  clk, rst_n, uart_rx (future), program_data[4:0]
Outputs (8): uart_tx, halted, guard_led, status[4:0]
Bidir (8):   debug/reserved
```

### Wishbone Register Map

| Offset | Register     | R/W | Description                              |
|--------|-------------|-----|------------------------------------------|
| 0x00   | CTRL        | W   | bit[0]: run, bit[1]: reset HCPU core     |
| 0x04   | STATUS      | R   | bit[0]: halted, bit[1]: guard, bit[2]: running |
| 0x08   | PROG_ADDR   | W   | Instruction address for loading (8-bit)  |
| 0x0C   | PROG_DATA   | W   | Instruction word (32-bit), writes to IMEM|
| 0x10   | GPR_SEL     | W   | Select GPR index for read-back           |
| 0x14   | GPR_DATA    | R   | Value of GPR[GPR_SEL]                    |
| 0x18   | CYCLE_CNT   | R   | Free-running cycle counter               |
| 0x1C   | FLAGS       | R   | Current flag register                    |
